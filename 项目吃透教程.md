## 一、项目速览

### 1.1 这是什么？

一个 **AI 驱动的旅游规划 Web 应用**（移动端 H5）。用户在手机上选择目的地、输入预算和天数，AI 自动生成一份完整的旅行计划——包括每日行程、景点介绍、交通方式、预算明细和旅行贴士。另外还有一个 AI 聊天页面，可以自由提问旅游相关问题，回答以流式打字机效果实时呈现。

### 1.2 核心功能

| 功能 | 描述 |
|------|------|
| **智能规划** | 输入城市+预算+天数 → AI 生成完整 JSON 行程 |
| **流式对话** | AI 聊天，逐字实时打字机效果 |
| **多 AI 切换** | 改 `.env` 一行配置，DeepSeek ↔ 硅基流动 随时切 |

### 1.3 技术栈一览

```
前端：Vue 3 (Composition API) + Vite + Vant UI + Vue Router + Axios
后端：Node.js + Express + LangChain.js + OpenAI 兼容接口
AI  ：DeepSeek (deepseek-chat) / 硅基流动 (DeepSeek-V4-Flash)
通信：RESTful JSON + SSE (Server-Sent Events) 流式推送
```

## 三、环境准备 & 启动

### 3.1 环境要求

- Node.js ≥ v18（项目开发用 v24.12.0）
- 一个能访问 DeepSeek 或硅基流动 API 的网络环境

### 3.2 安装依赖

```bash
# 在项目根目录执行（一键安装前后端所有依赖）
cd agent_travel_pr
npm install        # 安装 concurrently
npm run install:all  # 安装前端 + 后端依赖
```

### 3.3 配置 API Key

编辑 `travle-server/.env`：

```env
PORT=3300
MODEL_PROVIDER=DEEPSEEK            # 可选 DEEPSEEK 或 SILICONFLOW
SILICONFLOW_API_KEY=你的硅基流动key
SILICONFLOW_BASE_URL=https://api.siliconflow.cn/v1
SILICONFLOW_MODEL=deepseek-ai/DeepSeek-V4-Flash
DEEPSEEK_API_KEY=你的DeepSeek key
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
DEEPSEEK_BASE_MODEL=deepseek-chat
```

### 3.4 启动

```bash
# 在根目录一键启动前后端
npm run dev

# 等价于手动分别启动：
# 终端1: cd travle-server && npm run dev   (后端 3300 端口)
# 终端2: cd agentTravel && npm run dev      (前端 5173 端口，默认)
```

打开浏览器访问 `http://localhost:5173`

## 四、完整数据流：从点击到渲染

这是最重要的部分。跟着这个流程走一遍，项目的每个环节就都通了。

### 4.1 首页 → 详情页

```
[用户操作] 选择"北京"、预算"3000"、天数"3" → 点击"开始规划"
    │
    ▼
[Home/index.vue] handleSubmit()
    ├─ 前端校验：城市不为空、预算≥100、天数1~30
    └─ router.push({ path: '/detail', query: { city:'北京', budget:3000, days:3 } })
    │
    ▼
[Detail/index.vue] onMounted()
    ├─ 从 route.query 读取城市/预算/天数
    ├─ Number() 转换字符串为数字
    └─ 调用 fetchTripData()
    │
    ▼
[request.js] post('recommend', { city, budget, days })
    └─ axios 发送 POST 到 /api/travel/recommend
    │
    ▼
[vite.config.js] proxy 拦截 /api 前缀
    └─ 转发到 http://127.0.0.1:3300/api/travel/recommend
    │
    ▼
[travle-server/index.js] Express 接收请求
    └─ app.use('/api/travel', travelRouter) 路由匹配
    │
    ▼
[routes/travel.js] router.post('/recommend', async (req, res) => { ... })
    ├─ 解构 req.body 拿到 city/budget/days
    ├─ 参数校验（空值检查）
    └─ await travelService.recommend(city, budget, days)
    │
    ▼
[travelService.js] recommend(city, budget, days)
    ├─ 1. 参数校验：预算不能低于100，天数1~30
    ├─ 2. getTravelPrompt() 拼接提示词
    │      └─ 角色设定 + 参数 + JSON模板 + "只输出JSON"
    ├─ 3. this.llm.invoke(message) → 等待 AI 完整回复（非流式）
    ├─ 4. 三层正则提取 JSON：
    │      ├─ /```json\n([\s\S]*?)\n```/   (标准markdown)
    │      ├─ /```\n([\s\S]*?)\n```/       (无标注代码块)
    │      └─ /\{[\s\S]*\}/                (裸JSON)
    └─ 5. JSON.parse() → 返回 JS 对象
    │
    ▼
[routes/travel.js] try-catch 包裹，异常返回 500
    └─ res.json(result) → 把行程数据返回
    │
    ▼
[Detail/index.vue] fetchTripData() 收到 res
    ├─ res.success === true → tripData.value = res → 渲染行程
    ├─ res.error → errMsg.value = res.error → 显示错误
    └─ 兜底 v-else → 未知错误兜底
    │
    ▼
[模板渲染]
    ├─ 概览卡片：城市·天数 + 总预算
    ├─ van-collapse：每日行程折叠面板
    │   └─ 每行渲染 <SpotItem :data="day.morning|afternoon|evening" />
    ├─ <BudgetTable :data="budgetBreakdown" :total="totalBudget" />
    ├─ 温馨提示列表
    └─ 注意事项列表
```

### 4.2 聊天页 SSE 流式对话

```
[用户操作] 输入"北京有哪些必去景点？" → 点击发送
    │
    ▼
[Chat/index.vue] sendMessage()
    ├─ 添加用户消息到 messages[] → 模板渲染用户气泡
    ├─ 清空输入框
    └─ 调用 fetchAIResponse(msg)
    │
    ▼
[Chat/index.vue] fetchAIResponse()
    ├─ 创建空 AI 消息占位：{ id, role:'ai', content:'' }
    └─ 调用 fetchStream('chat', { message }, onChunk, onComplete, onError)
    │
    ▼
[request.js] fetchStream()
    ├─ fetch('/api/travel/chat', { method:'POST', body: JSON })
    ├─ response.body.getReader() 拿到可读流
    └─ while(true) 循环：
        ├─ reader.read() 读取一块数据
        ├─ TextDecoder 解码二进制 → 字符串
        ├─ buffer 缓冲区拼接待拼接（处理 TCP 分包）
        ├─ 按 \n 分割，逐行解析 SSE 协议
        ├─ data: 开头 → JSON.parse → onChunk(content)
        └─ event:end → onComplete()
    │
    ▼
[Chat/index.vue] onChunk 回调
    └─ 每收到一段文字：更新最后一条 AI 消息的 content
       → Vue 响应式 → 模板自动重新渲染 → "打字机效果"
    │
    ▼ (同时，后端)
[routes/travel.js] /chat 路由
    ├─ createStreamResponse(res) → 设置 SSE 响应头
    └─ await travelService.chat(message, callback)
    │
    ▼
[travelService.js] chat(message, streamCallback)
    ├─ SystemMessage('你是友好的旅游问答助手...')
    ├─ HumanMessage(用户消息)
    ├─ this.llm.stream(messages) → 异步迭代器
    └─ for await (const chunk of stream):
        └─ streamCallback(chunk.content) → 逐字推送给前端
    │
    ▼
[streamUtils.js] stream.send()
    └─ res.write(`data:${JSON.stringify({type:'chunk',content})}\n\n`)
    │
    ▼
[routes/travel.js] AI 全部回答完毕
    ├─ stream.send({type:'complete',content:result})
    └─ stream.end() → res.end()
```

## 五、逐个文件精讲

### 5.1 后端

#### `travle-server/src/index.js` — Express 入口

```js
app.use(cors())                     // 跨域（部署后可配置具体域名）
app.use(express.json())             // 解析 JSON 请求体 → req.body
app.use(express.urlencoded({extended:true}))  // 解析表单数据
app.get('/api/heartbeat', ...)      // 健康检查（给运维/监控用）
app.use('/api/travel', travelRouter) // 挂载旅游路由
app.listen(port, ...)               // 启动，监听 3300
```

**面试要点**：中间件的注册顺序很重要。CORS 必须最前面——如果请求被 CORS 拒绝，后面的 body 解析根本没机会执行。

#### `travle-server/src/routes/travel.js` — 路由层

两个接口：

| 路由 | 请求体 | 响应方式 | 说明 |
|------|--------|---------|------|
| `POST /api/travel/recommend` | `{city, budget, days}` | JSON（一次性返回） | 生成完整行程 |
| `POST /api/travel/chat` | `{message}` | SSE（流式推送） | AI 对话 |

`/recommend` 用 `try-catch` 包裹，异常返回 500。
`/chat` 也用 `try-catch`，异常通过 SSE 发送错误消息后关闭流。

**面试要点**：为什么路由层只做参数校验和响应，业务逻辑全在 Service？这是"关注点分离"——路由只负责 HTTP 相关的事（接收请求、返回响应），Service 负责业务逻辑。以后要换框架（如 Koa）只要改路由层。

#### `travle-server/src/services/travelService.js` — 核心服务

**单例模式**：`export default new TravelService()`，进程启动时初始化一次 AI 客户端，全局复用。

**构造函数**：
1. `this.llm = null`
2. `this.initLLM()` — 根据 `MODEL_PROVIDER` 环境变量选择 DeepSeek 或硅基流动

**`initLLM()`**：
```js
const provider = process.env.MODEL_PROVIDER  // DEEPSEEK 或其他
if (provider === 'DEEPSEEK') {
    apikey = process.env.DEEPSEEK_API_KEY
    baseURL = process.env.DEEPSEEK_BASE_URL
    model = process.env.DEEPSEEK_BASE_MODEL
} else {
    // 默认硅基流动
}
this.llm = new ChatOpenAI({
    configuration: { apiKey, baseURL, modelName: model },
    model,
    temperature: 0.5,   // 平衡创意和准确性
    streaming: true      // 支持流式，但 invoke() 不会逐字返回
})
```

**`recommend(city, budget, days)`**：
- `this.llm.invoke(message)` — 等待完整回复
- 三层正则提取 JSON
- 返回 JS 对象（非流式）

**`chat(message, streamCallback)`**：
- `this.llm.stream(messages)` — 异步迭代器逐字产出
- 每产出一段文字就调用 `streamCallback(content)`

**Prompt 设计（`getTravelPrompt`）**：
```
角色设定 → 参数填充 → 5条输出要求 → JSON模板 → "只输出JSON"
```

这是 Prompt Engineering 的核心：把"让 AI 自由回答"变成"让 AI 按模板填空"，大幅提高输出稳定性。

#### `travle-server/src/utils/streamUtils.js` — SSE 工具

```js
res.setHeader('Content-Type', 'text/event-stream')    // SSE 核心头
res.setHeader('Cache-Control', 'no-cache')             // 禁止浏览器缓存
res.setHeader('Connection', 'keep-alive')              // 保持连接
```

返回三个方法：
- `send(data)` → `res.write('data:' + JSON.stringify(data) + '\n\n')`
- `end()` → `res.write('event:end\ndata:{"done":true}\n\n')` → `res.end()`
- `error(msg)` → 发错误 → `res.end()`

#### `agentTravel/src/utils/request.js` — HTTP 工具

**Axios 实例**：
```js
const request = axios.create({
    baseURL: '/api/travel',    // Vite proxy 转发到后端
    timeout: 60000             // AI 生成最多等 60 秒
})
```
- 响应拦截器：`response.data` → 直接返回数据，调用方不用 `.data`
- `post(url, data)` + `get(url, params)` 封装

**`fetchStream()` — SSE 流式请求（核心）**：
- 用原生 `fetch`（Axios 不支持流）
- `response.body.getReader()` → 逐块读取
- `buffer` 缓冲区处理 TCP 分包（核心难点）
- 逐行解析 SSE 协议：`data:` 行 → JSON.parse → 回调 onChunk

**面试要点**：一定要能讲清楚 buffer 缓冲区的作用——网络传输中一个完整的 SSE 消息可能被 TCP 分成多个包，buffer 把不完整的最后一行暂存，等下一块数据到了拼接完整后再解析。

## 六、关键技术概念

### 6.1 LangChain.js 是什么？为什么不直接调 API？

LangChain.js 提供了：
1. **统一抽象**：`ChatOpenAI` 类兼容所有 OpenAI 接口格式的服务（DeepSeek、硅基流动、OpenAI 本家），换服务商只改配置
2. **消息管理**：`SystemMessage`（设定人设）、`HumanMessage`（用户输入），不用手动拼 `messages` 数组
3. **流式支持**：`invoke()` 阻塞 vs `stream()` 异步迭代器，一行代码切换

### 6.2 SSE vs WebSocket

| 特性 | SSE | WebSocket |
|------|-----|-----------|
| 方向 | 单向（服务器→客户端） | 双向 |
| 协议 | HTTP | ws://（独立协议） |
| 浏览器 API | EventSource（原生） | WebSocket（原生） |
| 复杂度 | 简单（纯文本流） | 较复杂（帧协议） |

本项目选 SSE：聊天场景只需要服务器推送 AI 回复，不需要客户端频繁发消息。SSE 更轻量、部署更简单（nginx 直接透传）。

**追问：为什么项目中用 fetch 而不是 EventSource？**
EventSource 只支持 GET 请求。我们需要 POST 把用户消息发给后端，所以手动用 fetch + ReadableStream 实现 SSE 协议解析。

### 6.3 TCP 分包与 buffer 缓冲区

```js
let buffer = ''
// 每收到一块数据：
buffer += decoder.decode(value)
const lines = buffer.split('\n')
buffer = lines.pop()  // 最后一行可能不完整，暂存
// 解析完整的行...
```

面试官如果问"流式传输最大难点是什么"——就讲这个。TCP 是面向字节流的，不保证消息边界。一个 `data:{"type":"chunk"...}\n\n` 可能被拆成两个 TCP 包到达，buffer 就是把不完整的最后一行保留下来等拼接。

### 6.4 Prompt Engineering 设计思路

从"让 AI 自由写"到"让 AI 填模板"：

1. **角色设定**："你是专业旅游规划师" — 限定 AI 的行为模式
2. **具体参数**："目的地${city}，预算${budget}元，${days}天" — 注入用户需求
3. **结构化要求**："必须包含：1... 2... 3... 4... 5..." — 列出强制内容
4. **模板约束**：给完整 JSON 模板，AI 只需填空 — 最大程度保证格式稳定
5. **反废话**："只输出JSON，不要额外解释" — 防止 AI 在 JSON 前后加废话

**追问：如果 AI 仍然不按模板输出怎么办？**
三层正则 fallback（代码块 → 无标注块 → 裸 JSON）+ 返回友好错误让用户重试。

### 7.2 准备一个 2 分钟项目介绍

> "我做的是一个 AI 驱动的智能旅游助手，移动端 H5 应用。前端用 Vue 3 + Vant UI，后端用 Node.js + Express，AI 部分用 LangChain.js 接入了 DeepSeek 和硅基流动两个大模型。
>
> 核心功能两个：一是输入目的地、预算、天数，AI 自动生成完整旅行计划——包括每日行程、景点详情、交通、预算明细和注意事项；二是一个 AI 聊天助手，可以自由提问，回答以流式打字机效果实时呈现。
>
> 技术上最有挑战的是流式对话的实现——我用 SSE 协议做服务端推送，前端手动解析 ReadableStream，用 buffer 缓冲区处理了 TCP 分包导致的消息截断问题。另外在 Prompt Engineering 上也花了不少功夫，用严格 JSON 模板加三层正则 fallback 来保证 AI 输出的稳定性。"
